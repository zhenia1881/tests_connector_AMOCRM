# tests_connector_AMOCRM

## Что находится в репозитории сейчас

На текущий момент репозиторий содержит минимальную заготовку:

- `.gitkeep` — пустой технический файл, который позволяет Git хранить пустой каталог в истории.

Иными словами, прикладного кода, конфигурации интеграции amoCRM, скриптов запуска и тестов пока нет.

## Как пользоваться этим репозиторием

### 1) Клонировать репозиторий

```bash
git clone <URL_репозитория>
cd tests_connector_AMOCRM
```

### 2) Проверить текущее состояние

```bash
ls -la
```

Ожидаемый минимальный набор в корне:

- `.git/`
- `.gitkeep`
- `README.md`

### 3) Начать наполнение проекта

Рекомендуемый порядок:

1. Добавить исходный код коннектора (например, `src/` или `app/`).
2. Добавить конфигурации окружения (`.env.example`, `config/`).
3. Добавить инструкции по запуску и тестам.
4. Добавить автоматические проверки (lint/test) в CI.

## Что можно получить и где это будет находиться

Пока проект пустой, из него можно получить только:

- **Историю Git** — в каталоге `.git/`.
- **Маркер наличия каталога в Git** — файл `.gitkeep` в корне.
- **Документацию по текущему состоянию** — файл `README.md` в корне.

После добавления кода обычно появляются следующие результаты:

- **Логи запуска** — часто в `logs/` или выводе терминала.
- **Отчёты тестов** — часто в `reports/`, `coverage/` или CI-артефактах.
- **Собранные артефакты** — часто в `dist/`, `build/` или `out/`.

> Сейчас этих директорий нет: они появятся после реализации соответствующих этапов.
